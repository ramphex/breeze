## Additional trust notes

- [`incidentJobs.ts`](/Users/toddhebebrand/breeze/apps/api/src/jobs/incidentJobs.ts) is not a BullMQ queue, but it explicitly uses `withSystemDbAccessContext()` and remains a system-level trust boundary.
- Agent-originated result jobs were prioritized first because they are the most realistic path for malformed or hostile payloads to enter Redis from a semi-trusted endpoint.
- Lower-privilege queues remain out of this pass by design. This document is intended to expand as those queue families are hardened in later passes.
